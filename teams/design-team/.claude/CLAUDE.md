# AI 设计团队协作系统

通过 SubAgent 实现角色隔离的设计团队协作。每个角色运行在独立上下文中，互不干扰。

- 角色间仅通过 `__ai__/` 目录下的文件通信
- 权限通过 PreToolUse hook 自动拦截越权操作
- 调度器在 agent 返回后执行 git diff 事后验证

# design-team 设计流程团队规则

## 角色调用规则

- 必须通过 `/角色名` 前缀命令显式调用角色，每次都需要带完整命令名
- 不支持隐式角色记忆，对话中不会自动延续上一次调用的角色上下文
- 角色运行在独立子进程中，互相隔离，仅通过文件通信

## 主对话行为（未调用角色命令时）

当用户**没有使用 `/角色名` 命令**，而是直接在对话中要求设计/修改原型等操作时，按以下优先级处理：

1. **只读操作直接执行**：阅读原型代码、分析现有设计、搜索样式、解释布局等不修改文件的操作，无需引导，直接执行
2. **小改动直接执行**：同时满足以下条件的改动，无需引导角色流程，直接执行：
   - 改动意图明确，不需要"先想清楚再动手"
   - 涉及 1-2 个文件
   - 不涉及新页面设计、信息架构变更、设计系统变更
   - 不是新功能的设计
   - 典型场景：修 CSS typo、调整单个颜色值、修正间距、修复响应式小问题
   - **拿不准时宁可引导，不要直通**
3. **设计行为先引导**：基于分析结果给出设计方案、用户流程建议、组件设计等属于**设计行为**，应引导到 `/产品设计师`
4. **原型制作先引导**：不满足第 2 条的 HTML/CSS 修改，建议使用对应角色（如"建议使用 `/UI设计师` 来制作原型，确保经过自检和审查流程"）
5. **用户坚持则执行**：**仅当**用户明确表示不走角色流程（如"不用角色"、"直接改"、"快帮我改了"）时，才直接执行，不再反复劝说。仅首次提醒"此操作不经过角色流程的质量保障"即可

## 累积感知与变更记录

对话中追踪直接执行的写操作次数和文件集合。当满足以下任一条件时触发提醒：

- 写操作达 **3 次**
- 涉及 **3 个文件**

触发后提供二选一：
- 继续执行，最后记录变更摘要
- 切换到对应角色命令

1-2 次且不超 2 文件的改动不触发。此规则仅适用于未调用角色命令的场景。

## 文件通信协议

角色间通过 `__ai__/design-team/tasks/<任务>/` 目录下的文件进行通信，不存在上下文泄漏。每个角色在自己权限范围内读写文件，其他角色通过读取这些文件获取上游信息。

## 文档模板

创建任何文档前，必须先读取 `.claude/templates/` 目录中的对应模板，按模板格式生成内容。

## 任务目录规范

任务目录命名格式：`YYYYMMDD_功能名_简短描述/`

示例：`20260410_用户登录_登录注册页设计/`

## 行为约束

- **不得修改** `.claude/` 目录下的任何文件
- **不得直接执行** `git push` 或 `git commit`（由用户手动操作）
- **不得修改** dev-team 或其他团队的任何文件（`__ai__/dev-team/` 等）
- **长对话提醒**：当同一任务累积 **2 次 rework**（即 status.md 变更记录中出现 2 次 `rework`），在下一次角色命令执行前提醒用户"当前任务已经历多轮返工，建议开启新对话继续，所有进度已保存在文件中，新对话可无缝衔接"

## 可用命令

| 命令 | 角色 | 用途 |
|------|------|------|
| `/产品设计师` | 资深产品设计师 & 信息架构师 | 需求分析、用户流程、设计规格、设计令牌 |
| `/UI设计师` | 资深 UI/UX 工程师 | HTML+CSS 原型制作（线框图 → 高保真原型） |
| `/设计审查员` | 首席设计质量架构师 | 设计一致性、可访问性、响应式、可用性审查 |

## 端到端流程示例

### 从零设计

1. **新设计任务**：`/产品设计师 描述你的设计需求`（创建设计规格 + 设计令牌 + 任务清单）
2. **制作原型**：`/UI设计师`（线框图 → 高保真 HTML+CSS 原型）
3. **设计审查**：`/设计审查员`（一致性、可访问性、响应式、可用性审查）
4. **修复审查问题**：`/UI设计师`（如有 rework）
5. **再次审查**：`/设计审查员`
6. **完成**：浏览器打开 mockup.html 查看最终设计

### 交接给 dev-team

设计完成后，dev-team 开发时可参考：
- `__ai__/design-team/tasks/{任务}/mockup.html` — 视觉原型
- `__ai__/design-team/tasks/{任务}/design-tokens.css` — 设计令牌
- `__ai__/design-team/tasks/{任务}/design-spec.md` — 设计规格

在 dev-team 的 `/项目经理` 设计时，design.md 中可写：
```
参考设计原型：__ai__/design-team/tasks/YYYYMMDD_功能名/mockup.html
参考设计令牌：__ai__/design-team/tasks/YYYYMMDD_功能名/design-tokens.css
```
