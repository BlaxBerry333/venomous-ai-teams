## 实证规则（硬约束）

- **路径用项目根相对**：`/` 开头 = 项目根（如 `/__ai__/web-design-team/designs/<slug>/...`、`/.claude/templates/...`）✅；裸路径 `__ai__/...` 会按用户当前 cwd 解析，用户从子目录跑命令会错位 ❌。Bash 命令例外（裸路径走真实 shell cwd）
- 优先把参数当 design-spec 路径，是路径先 Read 全文
- 模板拷贝：`Bash: cp -R .claude/templates/web-design-team/prototype-skeleton/. __ai__/web-design-team/designs/<dir>/prototype/`，已存在则跳过
- **依赖按需加**：模板默认仅 react + framer-motion + ts/vite。spec §8 声明用了 r3f / drei / three / gsap / lucide-react / @types/three 之一时，**第一步 npm install 之前**先 Edit package.json dependencies / devDependencies 加上对应包名 + 版本（@react-three/drei ^9.114.0 / @react-three/fiber ^8.17.10 / three ^0.169.0 / @types/three ^0.169.0 / gsap ^3.12.5 / lucide-react ^0.452.0）；**用了 r3f / three** 必须同时 Edit vite.config.ts 加 manualChunks 把 three 拆独立 chunk
- **占位资产硬规**：禁外链真实 CDN 图（除 https://picsum.photos / 免费 SVG 图标库 / shadcn 风格）；禁脑补真实视频 URL；优先 drei 几何体 / CSS 渐变 / lucide-react 图标 / 行内 SVG
- **动效实现**：完全照 spec §4 参数。spec 写 duration 0.8s power2.out 就写 0.8 + power2.out，不擅自改"更好看"的值
- **3D 场景**：照 spec §4 给的相机 / 光源 / 材质参数；spec 没给具体 shader 时优先 drei 现成（`<MeshDistortMaterial>` / `<MeshWobbleMaterial>` / `<Sparkles>`）禁自写 GLSL
- **可访问性 §6**：spec 写了哪个动效要 reduced-motion 降级就在该组件加 `useReducedMotion` 守卫
- 然后才写代码，禁凭印象写

## 步骤

1. 解析参数：design-spec 路径或裸需求
2. 裸需求 → 回："本 team 需 design-spec，请先 /web-design-team:设计师 <需求> 出稿"，**停**
3. Read design-spec 全文 + 模板骨架
4. Bash 创建 prototype 目录、拷模板
5. 按 spec §8 调整 package.json + vite.config.ts（见上方"依赖按需加"硬规）
6. 按 §2 组件树在 `prototype/src/components/` 拆文件，每个组件按 §4 对应动效节实现
7. 把设计 token §3 写到 `src/global.css` 顶部 `:root` CSS 变量
8. 占位资产按 §5 标注的实现方式落
9. 修 `src/App.tsx` 拼装组件
10. **build 守门**：`Bash: cd <prototype 目录> && npm install 2>&1 | tail -5 && npx tsc --noEmit 2>&1 | tail -10 && npx vite build 2>&1 | tail -10`。两类失败分别处理：
    - **tsc 错** → 真代码错（类型/导入），**必须改代码再 build，禁播报完成**
    - **tsc 过但 vite build 失败** → 先看错误：是代码问题（CSS 语法 / import 路径错 / 组件 export 错）→ 改代码；是环境/版本问题（peer dep 冲突 / 插件版本 / Node 版本） → **停下播报"build 因 <X> 失败，疑似环境/版本问题，建议主对话查证"，禁反复改代码越改越乱**
    - 两类都通过才进下一步
11. 重写 `prototype/README.md`：按本 design 内容生成（见下方"README 重写"节），禁留模板默认文案
12. 写 `prototype/placeholder-todo.md`：从 spec §5 资产清单逐行抄成 checklist（见下方"placeholder todo 模板"节），方便下游 team 接手时勾选
13. 完成后按调用模式播报：

    **`$ARGUMENTS` 含 `[调度模式]`**（被 `/web-design-team` 全流程调用，调度方会统一收口）→ 单行：「原型完成，路径：<...>。tsc / build 已通过。下游对接 checklist：prototype/placeholder-todo.md。」

    **`$ARGUMENTS` 不含 `[调度模式]`**（用户单跑 `/web-design-team:执行者`）→ 完整收口：
