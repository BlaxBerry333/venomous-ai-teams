# docs-team — 知识库文档专家团队

直接在主上下文中执行的领域专家。调用一次命令后，角色能力在当前对话中持续生效。

---

## 角色与命令

| 命令        | 角色               | 核心职责                                                  |
| ----------- | ------------------ | --------------------------------------------------------- |
| `/编程专家` | 编程开发知识库专家 | 创建/编辑/整理/审查编程开发相关的学习文档、VitePress 适配 |

后续可扩展：外语专家、运维专家等。

---

## 快速开始

```bash
cp -r teams/docs-team/.claude   your-project/
cp -r teams/docs-team/__ai__    your-project/

cd your-project && claude
# 输入: /编程专家 你的需求描述
```

### 典型用法

```
/编程专家 写一篇 React Hooks 的学习文档
        → 了解项目现状、参考现有风格、创建文档、更新 sidebar
继续写 useEffect 的高级用法
        → 角色粘性，无需重新调用命令
帮我整理 docs/javascript/ 目录下的文档
        → 全量阅读、评估现状、统一格式、保持文档间一致性
帮我审查下 docs/react/ 的内容质量
        → 5 维审查，输出报告，不直接修改
```

---

## 核心特性

### 角色粘性

调用 `/编程专家` 后，后续对话持续以该角色身份工作：

```
/编程专家 写 TypeScript 基础文档    ← 激活角色
继续写泛型那部分                    ← 无需再调命令
把重要知识点标注出来                ← 持续以编程专家身份工作
```

### 上下文质量保障

- **产出写文件，不留对话** — 长内容全部写入文件，对话中只输出摘要
- **操作前重读** — 每次修改前重新读取文件，不依赖上下文记忆

### 文档整体性保证

整理/编辑现有文档时：

- 修改概念定义 → grep 搜索所有文档中的引用，确保一致
- 修改示例代码 → 检查基于该示例的后续讲解
- 删除/重组章节 → 检查所有内部链接有效性
- 修改术语 → 全局统一更新

### 直接执行，不走子进程

与 dev-team 的 subagent 隔离模式不同，docs-team 的角色直接在主上下文中执行。一个对话中，一个专家，持续工作。

## 权限系统

### 角色权限矩阵

| 文件                        | 编程专家 |
| --------------------------- | :------: |
| 用户文档文件 (.md)          |    ✏️    |
| sidebar 配置文件            |    ✏️    |
| `__ai__/docs-team/**`       |    ✏️    |
| `.claude/**`                |    🚫    |
| 框架核心配置 (config.ts 等) |    🚫    |
| 其他团队文件                |    🚫    |

> ✏️ 可写 · 🚫 禁止

### 权限约束

Prompt 约束：command 中定义文件权限范围，禁止通过 Bash 写文件。settings.json deny 列表拦截破坏性操作。

## 文件结构

### 产出目录

```
__ai__/docs-team/
├── index.md          # 知识库总览（首次运行生成）
└── conventions.md    # 文档规范（格式、VitePress 配置、术语表）
```

### 安装后的用户项目

```
your-project/
├── .claude/
│   ├── CLAUDE.md            # 主对话规则（角色调用、行为约束）
│   ├── settings.json        # 权限配置（破坏性操作拦截）
│   └── commands/
│       └── 编程专家.md       # 编程开发知识库专家（直接执行）
├── __ai__/
│   └── docs-team/           # 团队产出目录
│       └── .gitkeep
├── docs/                    # 用户文档目录（VitePress 等）
└── (其他文件...)
```

## 当前支持的知识领域

编程开发全领域：

- 前端：HTML/CSS/JavaScript/TypeScript/React/Vue/Angular
- 后端：Node.js/Go/Java/Python/Rust
- 数据库：SQL/NoSQL/Redis/MongoDB
- 系统设计：分布式系统/微服务/消息队列
- 计算机基础：算法/数据结构/操作系统/网络协议
- 工程实践：设计模式/DevOps/CI-CD/Git

后续可扩展其他领域（新增 command 文件即可）。

## 配置

### 模型

角色默认继承用户当前模型。

### 语言

角色按用户使用的语言对话。文档内容语言根据同目录下的现有文档风格决定。
