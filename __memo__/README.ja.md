# 本リポジトリ開発メモ

[English](./README.md) | [中文](./README.zh.md) | 日本語

セッションを跨ぐ開発者メモ。新セッションはここを読む + `git log` で文脈を引き継ぐ、対話履歴に頼らない。

## ファイル命名

- `YYYYMMDD_短い説明.md`、例：`20260429_架构定稿.md`

## 執筆ルール

- 長さ自由（LLM コンテキストに入らない、token コストなし）、ただし無駄話禁止
- frontmatter `status:` 必須（中国語、引用符なし）、4 種類：
  - `进行中` — 未完了 todo あり；**SessionStart hook が未チェックの `- [ ]` 項目を対話に自動注入**
  - `已定稿` — 設計文書完成、ルール確定、永続的な参考価値あり（例：アーキテクチャ確定、共通規範）
  - `已完结` — 一回限りのイベント終了（例：特定の踏み抜き、特定のレビュー）
  - `暂存` — 意図的に保留、実際の痛みフィードバック待ちで判断（例：「未実装リスト」）
- **これがないと次回繋がらない**ものだけ書く：アーキテクチャ判断、踏み抜き教訓、判断根拠
- `git log` で分かることは書かない、**なぜ** だけ書く
- リスト + 短文、散文禁止

## 構造

ハードテンプレートなし。冒頭 1 行でなぜこの memo が存在するかを明示；中盤セクションは自由；未完了 todo がある場合は `## 本轮挂账` + `- [ ]` / `- [x]` チェックボックス。
